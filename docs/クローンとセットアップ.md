# 別のパソコンでクローンして動かす手順

別のPCでリポジトリをクローンし、開発サーバーを動かすまでの手順です。

---

## 1. 必要なもの

- **Node.js** … 20.x 以上を推奨（[nodejs.org](https://nodejs.org/)）
- **Git** … インストール済みであること
- **APIキー類** … 下記の環境変数用（Azure・Googleのポータルで取得）

---

## 2. クローン

```bash
git clone https://github.com/Behindthewisteria/-OCR.git
cd -OCR
```

（SSHでクローンする場合: `git clone git@github.com:Behindthewisteria/-OCR.git`）

---

## 3. 依存関係のインストール

```bash
npm install
```

---

## 4. 環境変数の設定

**`.env.local`** をプロジェクト直下に作成します。  
雛形として **`.env.example`** があるので、コピーして値を埋めてください。

```bash
cp .env.example .env.local
# .env.local を編集して各キーを設定
```

（`.env.local` は `.gitignore` で除外されているため、リポジトリには含まれません。**別PCでは毎回作成が必要**です。）

```bash
# Azure Document Intelligence（OCR）
AZURE_DI_ENDPOINT=https://xxxxxxxx.cognitiveservices.azure.com/
AZURE_DI_KEY=あなたのキー

# Azure Custom Vision（記号識別）
NEXT_PUBLIC_CUSTOM_VISION_KEY=あなたのキー
NEXT_PUBLIC_CUSTOM_VISION_ENDPOINT=https://xxxxxxxx.cognitiveservices.azure.com/
NEXT_PUBLIC_CUSTOM_VISION_PROJECT_ID=プロジェクトID
NEXT_PUBLIC_CUSTOM_VISION_ITERATION_ID=イテレーションID

# Google Gemini API（文書情報・メニュー検証）
GEMINI_API_KEY=あなたのAPIキー
```

- **AZURE_DI_*** … Azure Portal → Document Intelligence リソースの「キーとエンドポイント」
- **Custom Vision** … Azure Custom Vision の「パフォーマンス」→「予測URL」などから取得（Prediction-Key / Endpoint / Project ID / Iteration ID）
- **GEMINI_API_KEY** … [Google AI Studio](https://aistudio.google.com/) でAPIキーを発行

既存のPCで動いている場合は、その **`.env.local` の内容をコピー**して新しいPCの `.env.local` に貼り付けると確実です（キーは共有可能な前提で）。

---

## 5. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで **http://localhost:3000** を開きます。

- 未ログインの場合は `/login` にリダイレクトされます。
- ログイン: ユーザー名 `admin` / パスワード `admin`（コード内で固定）

---

## 6. よくあるトラブル

| 現象 | 対処 |
|------|------|
| ログインできない | ユーザー名・パスワードは `admin` / `admin`。Cookieが無効になっていないか確認。 |
| 解析で 500 エラー | `.env.local` が存在するか、`AZURE_DI_ENDPOINT` / `AZURE_DI_KEY` が正しいか確認。 |
| 記号が判定されない | Custom Vision の4つ（KEY, ENDPOINT, PROJECT_ID, ITERATION_ID）がすべて設定されているか確認。 |
| Excel出力でエラー | `public/templates/請求書テンプレート.xlsx` が存在するか確認。テンプレートはリポジトリに含まれている必要があります。 |

---

## 7. まとめ（チェックリスト）

1. リポジトリを **クローン**
2. **`npm install`**
3. **`.env.local`** を作成し、必要な環境変数を設定（既存PCからコピー可）
4. **`npm run dev`** で起動
5. ブラウザで **http://localhost:3000** → ログイン（admin / admin）して利用

環境変数さえ揃っていれば、どのPCでも同じ手順で動かせます。
